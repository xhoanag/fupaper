\section{Evaluation Approach}

\subsection{Data Basis}
\label{data_basis}
A data-driven approach was used to derive and synthesize use cases from authentic learning interactions. In educational settings, evaluations must reflect how learners and instructors actually use systems, as practical applicability determines their value. Grounding use cases in real study contexts enables assessment across relevant scenarios and strengthens external validity.

The analysis started from approximately $13,000$ real prompts extracted from the productive chatbot logs. After rule-based cleaning, a filtered dataset of $11,778$ prompts remained and was used for subsequent modeling steps (see \autoref{tab:data-processing} for the full pipeline and stage-wise counts).
The cleaned prompts were embedded in a multilingual semantic space and clustered to identify recurring interaction patterns. Because cluster outputs are not always directly interpretable as practical scenarios, they were manually consolidated into seven high-level use case categories.

From these categories, three representative use cases were selected for the evaluation to ensure both coverage and comparability. (1) Technical support represents the largest cluster group and forms the first use case. It encompasses organizational and structural platform questions, with typical examples being \enquote{How much does a course cost?}, \enquote{Where can I find certificates of attendance?}, or \enquote{Who are the contact persons?}. The second and third use cases, (2) course-related subject questions and (3) learning-oriented interactions, are derived from the second-largest cluster group. They address domain-specific inquiries during an active learning situation, often tied to a specific system context like a Moodle course. Example prompts include \enquote{How does an LLM work?}, \enquote{What is sklearn?}, or \enquote{What is the difference between Deep Learning and Machine Learning?}. Unlike technical support, these scenarios demand a deeper context integration; the system must correctly identify course-internal materials and synthesize them in a structurally coherent manner. 

Real user prompts from these largest clusters were extracted and utilized across all subsequent evaluation approaches, whereas prompts from the smaller cluster groups (e.g., polite phrases) were neglected for the evaluation.

\subsection{Multidimensional Comparative Evaluation Approach}

\paragraph{1) RAGAS: Automatic Evaluation Along the RAG Dimensions}

For a scalable and systematic assessment, we employed the RAGAS \autocite{es2025ragas}. This framework automatically evaluates RAG pipelines along key dimensions: retrieval effectiveness, generation quality, and factual correctness. RAGAS is particularly suited for applied test settings, as it accommodates large volumes of heterogeneous queries without requiring manually created reference answers, which are often infeasible to obtain.

RAGAS operationalizes evaluation along the RAG dimensions by providing automated metrics that quantify the effectiveness of individual RAG components \autocite{es2025ragas}. In this study, the metrics \textit{Answer Relevancy}, \textit{Context Relevance}, and \textit{Faithfulness} were used. \textit{Answer Relevancy} measures the semantic alignment between the user query and the generated response. \textit{Context Relevance} evaluates whether the retrieved context segments are suitable for answering the query and therefore reflects the effectiveness of the retrieval step. \textit{Faithfulness} assesses whether the generated answer is supported by the provided context and does not contain unsupported or hallucinated information. Together, these metrics provide a quantitative approximation of retrieval quality, generation quality, and factual correctness.

The evaluation was based on a curated set of 60 questions representing the use cases of technical support queries and course-related subject questions. For each evaluation instance, the user query, the generated answer, and the context segments used during answer generation were passed to the RAGAS framework. This ensured that the evaluation reflected the actual system behavior rather than an idealized retrieval output and enabled a direct comparison between system versions. To reduce the influence of stochastic effects in LLM-based evaluation, each question was processed five times and the resulting metric values were averaged. The scoring itself was performed using a separate large language model acting as the evaluation model, ensuring consistent scoring across all samples.

While the automated RAGAS evaluation provides a quantitative view of retrieval effectiveness, answer relevance, and factual grounding, it cannot fully capture pedagogical quality, as it does not account for the third use case, or perceived usefulness, which are particularly relevant in educational applications. Therefore, RAGAS was used as the core component of the automatic evaluation and complemented by additional evaluation methods within the overall evaluation approach.

\paragraph{2) Expert blind assessment: Gaining insights into perceived quality and educational value}

To complement automated evaluation, an expert-based blind test was conducted to comparatively assess the two RAG systems. The evaluation followed an A/B design in which experts were presented with two system responses (A/B) per prompt in an identical interface, without knowledge of the underlying system configuration. This setup is inspired by arena-style evaluation approaches commonly used for generative models, which aim to reduce expectation and confirmation bias in human judgments \autocite{khan2025ragbenchmark}.

The study was conducted as moderated one-on-one sessions with a duration of approximately 30 to 60 minutes per participant. In total, 14 experts participated, including authors of e-learning content, educators, and domain experts from applied research and academic institutions. The evaluation followed a standardized procedure: First, participants were introduced to the evaluation setup and rating criteria. Next, the synthesized use cases were presented together with curated prompts derived from historical user queries. For each prompt, two system outputs were displayed in randomized order (\autoref{fig:expert_blindtest}). Participants evaluated both responses independently using a five-point Likert scale (1 = strongly disagree, 5 = strongly agree). In addition, participants were asked to verbalize their reasoning in a think-aloud setting, which was recorded and transcribed for qualitative analysis.

The evaluation combined quantitative and qualitative assessment to enable a comprehensive comparison of system performance. Quantitatively, responses were rated across multiple dimensions, including content correctness, completeness/efficiency, relevance, contextual integration/coherence, clarity/structure, and learning support/didactics (\autoref{tab:expert_evaluation_dimensions}). The evaluation statements for these dimensions were conceptually derived from established criteria by \autocite{neumann2025llm} and \autocite{hmoud2024rubric}. These criteria capture both technical answer quality and pedagogical effectiveness, which are particularly relevant in educational chatbot settings.

Qualitatively, the think-aloud transcripts provide insights into expert decision-making processes, perceived strengths and weaknesses, and domain-specific expectations. This mixed-methods approach allows not only for a systematic comparison of both systems but also for a deeper understanding of how differences in system design affect perceived answer quality.


\subparagraph{3) User blind assessment: Gaining quantitative insights}

In contrast to the expert interviews, which provide in-depth, qualitative insights from domain specialists, the Chatbot Arena captures aggregated preference signals from a broader user group under standardized conditions. This allows for a more behavior-oriented assessment of perceived response quality, complementing the expert-driven analysis with scalable, user-centered evidence. Together, both approaches enable a more comprehensive evaluation by combining structured judgment with real user perception.

    \label{fig:user_blindtest}
    \vspace{-0.8em}
\end{wrapfigure}

The Chatbot Arena, depicted in \autoref{fig:user_blindtest}, was designed as a controlled evaluation platform for pairwise model comparisons and is methodologically based on the principles of the  Large Model Systems Organization (LMSYS) Chatbot Arena \autocite{Zheng2023LMSYSArena}. The goal is not free interaction, but rather the standardized evaluation of response quality under controlled conditions. To this end, a blind A/B setup is used. Participants see two responses to the same question and choose between \textit{A}, \textit{B}, \textit{tie}, or \textit{both bad}. Pairwise preference comparisons are considered a robust evaluation method, as relative judgments are cognitively more consistent than absolute ratings \autocite{Thurstone1927ComparativeJudgment, BradleyTerry1952PairedComparisons} and generate an aggregatable preference signal. For experimental control purposes, the 60 questions in the RAGAS evaluation were divided into four subsets. Responses are dynamically generated for each session to ensure realistic conditions. Assignment is balanced using a round-robin procedure, ensuring even load distribution and sample representation \autocite{Dillman2014TailoredDesign,Tourangeau2000SurveyResponse}. At the same time, this division reduces fatigue; approximately 15 questions are evaluated per participant. Additionally, the order of the answers is randomized to minimize presentation bias and ensure internal validity \autocite{Kohavi2020TrustworthyExperiments,Joachims2017UnbiasedLTR}.

\paragraph{4) Cost Analysis: Overhead of Architectural Choices}
Cost efficiency has emerged as a critical dimension in both best-practice guidelines and recent evaluation literature on RAG systems, particularly in the context of real-world deployments. While much prior work focuses on accuracy and retrieval quality, productive systems must additionally satisfy constraints related to computational cost and responsiveness. This is especially relevant for educational platforms, where high interaction volumes and sustained user engagement amplify even small inefficiencies in token consumption or latency.

Accordingly, this study incorporates a cost analysis that captures both economic and performance-related aspects of system behavior. Token usage is used as a direct proxy for monetary cost in API-based LLM deployments. To ensure transparency and reproducibility, all measurements are based on a consistent pricing scheme using the \textit{gpt-4o} model (Azure, region \textit{swedencentral}, version 2024-05-13), with costs of $€4.18621$ per million input tokens and $€13.8145$ per million output tokens (numbers obtained February 2026). This allows precise estimation of per-query costs and facilitates direct comparison between system variants. Complementarily, latency metrics—specifically Time-To-First-Token and total response time—capture system responsiveness and perceived interaction quality. By jointly analyzing token consumption and latency, the evaluation accounts for trade-offs introduced by architectural enhancements such as routing or reranking. This enables a holistic assessment that aligns technical performance with economic feasibility in practical deployment scenarios.
